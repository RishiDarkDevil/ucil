**W-2 — Weak assertion in `all_supported_languages_load_without_error`**

`crates/ucil-treesitter/src/parser.rs:234–241`:

```rust
assert!(
    result.is_ok() || matches!(result, Err(ParseError::ParseFailed { .. })),
    "language {lang:?} produced unexpected LanguageLoad error"
);
```

The `|| ParseFailed` arm is effectively dead code for the chosen test input `"// test"`, because tree-sitter never returns `None` (→ `ParseFailed`) for a syntax-invalid input — it returns a tree with error nodes. The assertion is correct at gating `LanguageLoad` errors, which matches the intent, but allows a genuine `ParseFailed` to pass silently if it ever occurred. A stricter `assert!(result.is_ok(), ...)` would be more defensive. Not a blocker (the logic matches the stated intent and no tree-sitter grammar legitimately returns `None` on this input), but a future-proofing concern.

**W-3 — `SessionError` not re-exported from `ucil-daemon/src/lib.rs`**

`crates/ucil-daemon/src/lib.rs:11` re-exports only `{SessionId, SessionInfo, SessionManager, WorktreeInfo}`. All three public async methods on `SessionManager` return `Result<_, SessionError>`, so callers that need to match error variants must use the full path `ucil_daemon::session_manager::SessionError`. The WO spec does not list `SessionError` in the re-export list (so the executor followed spec), but the public API is ergonomically incomplete. A follow-on commit or the next WO touching `ucil-daemon` should add `SessionError` to the `pub use` line.
