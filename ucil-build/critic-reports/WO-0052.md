## Notes for the verifier

- **Order-preservation invariant.** SA3 expects `[b, e]` (input order:
  `[a, b, c, d, e]` minus in-context `{a, c, d}`). The `filter` chain
  preserves the input Vec's iteration order — confirmed at
  `crates/ucil-daemon/src/session_manager.rs:313-316`
  (`candidates.into_iter().filter(...).collect()`). If a future refactor
  routes through `BTreeSet::difference` or similar set-arithmetic, SA3's
  `[b, e]` expectation would still hold by lexical accident
  (`a < b < c < d < e`) — but the order-preservation property documented
  in the rustdoc would silently regress. Verifier may want to spot-check
  via a candidate Vec that is NOT in lexical order (e.g. `[e, b, a]`)
  before flipping.

- **`set_ttl(1)` + `purge_expired(created_at + 2)` is a wall-clock-free
  cycle.** SA5 does NOT depend on real time elapsing — it constructs the
  `now_secs` argument to `purge_expired` directly from the captured
  `created_at + 2`. So the test is deterministic under stress / parallel
  execution / coverage instrumentation. No `tokio::time::sleep`
  anywhere in the body.

- **Doctest `no_run` is harmless to AC04 and AC11.** AC04
  (`cargo test -p ucil-daemon session_manager::test_session_dedup`)
  targets the `#[tokio::test]` function specifically, not doctests. AC11
  (`cargo doc -p ucil-daemon --no-deps`) does not execute doctests,
  only renders them. AC08 (`cargo test --workspace --no-fail-fast`)
  WOULD execute doctests by default — but `no_run` suppresses execution
  there too. So Warning #2 has no effect on any AC.

- **Marker file (Warning #1) blocks AC13 only.** If the executor commits
  `ucil-build/work-orders/0052-ready-for-review.md` between this critic
  report and the verifier session, the AC13 diff allow-list will pass.
  The marker is conventionally written by the executor as the last
  pre-verifier step; absence here likely just means the executor has not
  yet performed that step. Verifier should confirm the marker exists
  before running the AC13 grep, or re-trigger the executor with a
  marker-write nudge.
