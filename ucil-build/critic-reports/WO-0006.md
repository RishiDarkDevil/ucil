### Warnings (should fix before gate)

#### W1. Refactor commit `705e5d6` omits `Feature: P1-W2-F03` trailer

`refactor(treesitter): fix all clippy::pedantic warnings in symbols + chunker`
modifies both `crates/ucil-treesitter/src/symbols.rs` (P1-W2-F02) and
`crates/ucil-treesitter/src/chunker.rs` (P1-W2-F03), but carries only
`Feature: P1-W2-F02` in its trailer. Commit style requires every
feat/fix/refactor touching a feature to carry the relevant `Feature:` trailer.
Missing: `Feature: P1-W2-F03`.

#### W2. `ChunkError::ParseRequired` is unreachable dead code

`crates/ucil-treesitter/src/chunker.rs` — `ChunkError::ParseRequired` is
defined but `Chunker::chunk` always returns `Ok(...)`. The executor documented
this as a "future API variant", but dead error variants are noise and may fail
clippy's `dead_code` lint depending on configuration. Recommend either:
- removing it until actually needed (per UCIL anti-speculative-abstraction
  rule), or
- adding a `#[doc(hidden)]` + `#[allow(dead_code)]` with a `// TODO(WO-???)`
  reference; or escalate with an ADR explaining the forward-compat rationale.
