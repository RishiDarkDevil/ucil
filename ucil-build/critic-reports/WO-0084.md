### Warnings (should fix before gate, OR verifier should consider)

1. **Subtle correctness fragility in `find_observed_at` helper**
   `crates/ucil-core/src/tier_merger.rs:310-343` — the private helper recovers
   a winning observation's `observed_at` by linear-scanning the input slices for
   the FIRST `(value, tier, confidence)` triple match. The helper's inline
   comment asserts "the FIRST such observation in tier-order [...] is also the
   recency winner (the merge updates `source_tier` + `confidence` monotonically
   toward the recency winner)" — but this monotonicity claim only holds when
   the input slices contain at most one observation per `(value, tier,
   confidence)` triple. If a real-world tier source emits two observations of
   the same value within the same tier carrying the same confidence at
   different timestamps (`T_old`, `T_new` with `T_old < T_new`), the helper
   returns `T_old`, and a subsequent cross-tier observation at `T_middle`
   (`T_old < T_middle < T_new`) would be erroneously declared the new recency
   winner — overwriting the actually-fresher `T_new` entry. The frozen
   `test_multi_tier_query_merge` does not exercise this edge case (each
   scenario uses unique `(tier, confidence)` triples per value), so the bug
   does not surface today. **Not a blocker** — the WO acceptance is satisfied
   and the `T: Clone + PartialEq` bound is the documented dependency-inversion
   contract — but verifier should note this as a future-correctness landmine
   for the consumer WO that wires real tier sources. A defensive fix would be
   to track `observed_at` directly on `MergedResult` or as a side-table keyed
   by index, rather than recovering it via re-scan.

2. **Disclosed M1/M2 panic-SA deviations** (informational, accepted by
   precedent) — the executor's RFR transparently discloses two
   spirit-over-literal deviations vs `scope_in #15-#16`:
   - M1 panic surfaces on SA3 first (per assertion order) rather than the
     prescribed SA6 — both SAs are authority-precedence inversions, the
     mutation contract is substantively satisfied. Per WO-0070/0083
     spirit-over-literal precedent. ✓
   - M2 panic surfaces on SA2 first rather than the prescribed SA3 or SA5
     — the mutation also flips SA3 + SA5; the test runner halts on the
     first failure (SA2) by Rust convention. Substantively satisfied. ✓

   These are explicitly documented in the RFR's `## Disclosed deviations`
   section. Not a blocker; verifier accepts per the established 14-WO-deep
   precedent. Flagged for completeness only.

3. **Feat commit at 995 LOC vs the 50-line soft target** — the commit
