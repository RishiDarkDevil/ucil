### Warnings (informational — do not block verifier)

1. **Args shape deviates from WO scope_in #3 wording — but is correctly
   justified**. The WO scope_in #3 originally proposed
   `args=['--from', '<git+ref>', '<entry-point-name>']`. The executor
   discovered the upstream `mcp_server/pyproject.toml` does NOT declare a
   `[project.scripts]` console-script entry point, so a bare
   `graphiti_mcp_server` would fail. The args are instead
   `['--from', '<git+ref>', 'python', '-m', 'graphiti_mcp_server',
   '--transport', 'stdio']` — a `python -m` invocation of the importable
   top-level module. The deviation is documented at
   `plugins/knowledge/graphiti/plugin.toml:18-32` in the leading comment
   block and in the c5aac51 commit body. This is a sensible adaptation
   to upstream packaging reality, not a scope violation; flagging only so
   the verifier sees the rationale is captured in writing.

2. **Test PASSes by skip-early on a clean session without env vars**.
   `graphiti_manifest_health_check` returns early via
   `graphiti_skip_via_env()` (`crates/ucil-daemon/tests/g3_plugin_manifests.rs:216`)
   when `FALKORDB_URI`/`NEO4J_URI` is unset OR when no LLM API key is set.
   This is the documented two-gate pattern (WO-0079 scope_in #5(iii) /
   #6 / #20), and the verifier MUST NOT set the
   `UCIL_SKIP_KNOWLEDGE_PLUGIN_E2E` opt-out (scope_in #25). The
   load-bearing assertion is conditional on operator-state env presence;
   the M1 mutation contract was verified live with env vars set per the
   RFR (`ucil-build/work-orders/0079-ready-for-review.md:74-96`).
   Verifier note (already in the RFR): to actually exercise the spawn
   path during verification, set FalkorDB + a placeholder
   `OPENAI_API_KEY`. Otherwise the test trivially returns early — same
   as the existing two G3 tests and the WO-0069 precedent.
