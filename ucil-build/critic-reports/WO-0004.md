#### 1. Missing `tokio::time::timeout` on async IO await — `verify_plugin_health()`

**File**: `crates/ucil-cli/src/commands/init.rs:178–182`

```rust
// lines 178-182
let kind = match tokio::process::Command::new(bin)
    .arg("--version")
    .output()   // ← .await below, no timeout
    .await
{
```

The `.output().await` for each P0 binary probe has **no `tokio::time::timeout` wrapper**.

This violates the project-wide invariant in `CLAUDE.md`:

> All async code uses `tokio::time::timeout` on any await that touches IO.

If any of the six P0 binaries (`serena`, `rust-analyzer`, `pyright`, `ruff`, `eslint`,
`shellcheck`) takes an unexpectedly long time to respond to `--version` (e.g. a hung
socket wrapper), `ucil init` will hang indefinitely. The fix is a per-binary timeout
constant, e.g.:

```rust
const PLUGIN_PROBE_TIMEOUT: Duration = Duration::from_secs(5);

let output = tokio::time::timeout(
    PLUGIN_PROBE_TIMEOUT,
    tokio::process::Command::new(bin).arg("--version").output(),
)
.await;
let kind = match output {
    Ok(Ok(out)) if out.status.success() => PluginStatusKind::Ok,
    _ => PluginStatusKind::Degraded,
};
```


`rust-style.md` specifies: *"cargo nextest is the default runner."*
The CI workflow uses `cargo test --workspace` instead of
`cargo nextest run --workspace`. The WO implementation notes happen to say
`cargo test`, but the project standard diverges. Nextest provides per-test
timeouts, better output, and retry support. Consider aligning CI with the
project's default runner.

#### W4. `.github/workflows/ci.yml:74` — `uv sync --all-extras` vs. `--all-packages`

```yaml
- name: uv sync
  run: uv sync --all-extras    # line 74
  working-directory: ml
```

The WO implementation notes specify `uv sync --all-packages`. These flags have
different semantics: `--all-extras` installs optional dependencies of the root
package; `--all-packages` syncs every package in a uv workspace. If `ml/` is a
uv workspace with multiple packages, `--all-extras` may skip them. Verify that
`--all-extras` is intentional here.
