## Suggested remedial action (planner / user)

**Option 2 (recommended)** — additive `detect_watchman_in` helper via
`which::which_in`, sidestepping `std::env::set_var("PATH", …)` in the
four PATH-mutating watcher tests. Blast radius ~15 source lines:

```rust
// crates/ucil-daemon/src/watcher.rs

pub fn detect_watchman() -> Option<WatchmanCapability> {
    detect_watchman_in(std::env::var_os("PATH"))
}

#[must_use]
#[tracing::instrument(name = "ucil.daemon.watcher.detect_watchman_in", level = "debug")]
pub fn detect_watchman_in(search_path: Option<std::ffi::OsString>) -> Option<WatchmanCapability> {
    let cwd = std::env::current_dir().ok()?;
    which::which_in("watchman", search_path, &cwd)
        .ok()
        .map(|binary| WatchmanCapability { binary })
}
```

Then update the four tests
(`test_watchman_detection`,
`test_auto_select_backend_returns_notify_when_watchman_absent`,
`test_auto_select_backend_returns_watchman_when_available_and_above_threshold`,
`test_auto_select_backend_returns_notify_when_below_threshold`) to
call `detect_watchman_in(Some(shim_dir.as_os_str().into()))` instead
of mutating the process env. Delete the `ENV_GUARD` / `RestorePath`
scaffolding. `auto_select_backend` gains a companion
`auto_select_backend_with_path` with the same extension, or takes the
optional `search_path` parameter directly.

The frozen F03 selector name `watcher::test_watchman_detection`
stays. The public zero-arg `detect_watchman()` stays (still reads
`PATH` from env via `var_os`). No ADR required — DEC-0005 untouched,
master-plan §2 line 138 untouched; `which::which_in` is already
available in the workspace (`which` 7.0.3 in `Cargo.lock`).
