## Failure detail — P1-W2-F06 selector mismatch

**Criterion**: `cargo nextest run -p ucil-daemon storage::test_two_tier_layout`

**Expected**: 1 test runs and passes  
**Actual**: exit code 4, "0 tests run: 0 passed, 8 skipped", "error: no tests to run"

**Root cause**: The test is declared inside `#[cfg(test)] mod tests { ... }` in
`crates/ucil-daemon/src/storage.rs:174-231`. This makes the test's nextest path
`storage::tests::test_two_tier_layout`. The frozen acceptance test selector in
`feature-list.json` and the work-order both specify `storage::test_two_tier_layout`
(no `tests::` component), which matches nothing.

```
$ cargo nextest run -p ucil-daemon storage::test_two_tier_layout --test-threads 4
    Finished `test` profile [unoptimized + debuginfo] target(s) in 0.04s
────────────
 Nextest run ID 3ab7fcd6-... with nextest profile: default
    Starting 0 tests across 2 binaries (8 tests skipped)
────────────
     Summary [   0.000s] 0 tests run: 0 passed, 8 skipped
error: no tests to run
(hint: use `--no-tests` to customize)
```

The test itself is functionally correct and passes when invoked with the full path:
```
$ cargo nextest run -p ucil-daemon "storage::tests::test_two_tier_layout"
    PASS [   0.003s] ucil-daemon storage::tests::test_two_tier_layout
```

**Required fix**: Remove the `mod tests { ... }` wrapper in `storage.rs` and
place test functions directly in the `#[cfg(test)]` scope at module root, e.g.:

```rust
#[cfg(test)]
#[test]
fn test_two_tier_layout() { ... }
```

This makes the nextest path `storage::test_two_tier_layout`, matching the frozen
selector. (The frozen `acceptance_tests` selector cannot be changed.)
