### Note on `scripts/verify/coverage-gate.sh`

The script exits 1 with `line=51%` in the current shell environment.
This is the same pre-existing harness-infrastructure bug first
documented in the WO-0039 retry-1 verifier report (2026-04-19T01:34Z)
and carried through WO-0040 verifier's PASS verdict
(2026-04-19T02:30Z). Root cause is twofold:

1. `RUSTC_WRAPPER=sccache` prevents instrumentation from reaching every
   test binary — the script-driven invocation only records main.rs
   (`lines.count=249` vs. the crate's real 5,372 instrumented lines).
2. The script's `-size 0 -delete` prune misses 472-byte corrupt
   profraw files written by detached watcher/daemon threads.

Documented in escalation
`ucil-build/escalations/20260419-0152-monitor-phase1-gate-red-integration-gaps.md`.
Not a WO-0041 regression. WO-0041's primary new code (`src/main.rs`)
is at **94.78% line / 100% function** coverage under the workaround.
