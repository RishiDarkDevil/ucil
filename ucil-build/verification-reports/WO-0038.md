## Verifier-side observation — criterion 1 regex bug

For the planner's reference: nextest v0.9.x emits the summary line
exactly as `<N> tests run: <M> passed[, <K> skipped][, <L> failed]`. When
`M == N` and the run is all-green, there is no second `<digits> passed`
token in the line, so `.*[0-9]+ passed` cannot match after the leading
capture group. Suggested replacement:

```
grep -E 'tests run: ([5-9]|[1-9][0-9]+) passed' /tmp/nextest-WO-0038.log
```

This would make the criterion self-consistent with the tests-run count
while still failing on an all-skipped or all-failed run. An ADR codifying
the nextest-summary pattern (analogous to DEC-0007 for coverage) would
help future WOs avoid the same mistake.
