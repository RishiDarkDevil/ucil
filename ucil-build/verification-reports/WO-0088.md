### §B — M3 first-failing SA is SA2 (not SA3 / SA4 as enumerated)

Per scope_in #16 third paragraph + RFR §(d)2: M3's logical inversion
of the attachment-threshold comparison flips ALL THREE of SA2 / SA3 /
SA4. Rust halts on first failure, and SA2 is asserted first in the
test body (`(SA2) top-hit bonus expected Some(...); observed None`),
so it surfaces before SA3 / SA4. AC36 reads "FAILS the cargo-test
selector with one of the SA2/SA3/SA4 panic bodies" — SA2 satisfies it.
This is the planner's anticipated outcome and the same spirit-over-
literal carve-out applied in WO-0070 / WO-0083 / WO-0084 / WO-0086.

### §C — `cargo test ... -- --list` separator

The work-order's `acceptance_criteria` indices 26 / 27 use the form
`cargo test -p ucil-core ... --list`, but recent Cargo (1.94+)
requires the `-- --list` separator. Verifier used the corrected form
(same as WO-0087 verifier-acknowledged carry-forward) and confirmed
the count `1` is identical between forms.

### §D — M1 alternate-form (constant-flip prepend, not closure-body replacement)

The literal form prescribed in scope_in #16 ("change the closure body
to `false`") would remove both `hit` and `files_in_context` from the
closure body, failing to compile under the workspace's
`#![deny(warnings)]` lint-level. The applied alternate-form prepends
`false &&` to the dedup conjunction: still constant-flips the branch
to "always not deduped", still trips SA2 cleanly, but preserves
variable references so the file compiles. Same spirit-over-literal
carve-out applied in WO-0070 / WO-0083 / WO-0084 / WO-0086 / WO-0085
W2 lessons; disclosed in the executor's RFR §(d)1.
