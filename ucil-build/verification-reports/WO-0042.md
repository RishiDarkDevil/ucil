### Note on `scripts/verify/coverage-gate.sh`

The script exits 1 in the current shell environment. This is the same
pre-existing harness-infrastructure bug carried through WO-0039
retry-1, WO-0040, and WO-0041 verifier reports. Root cause: (1)
`RUSTC_WRAPPER=sccache` prevents instrumentation from reaching every
test binary; (2) detached subprocess threads write 472-byte
corrupt-header profraw files that the script's zero-byte prune misses.
Documented in escalation
`ucil-build/escalations/20260419-0152-monitor-phase1-gate-red-integration-gaps.md`.
The `env -u RUSTC_WRAPPER cargo llvm-cov` workaround restores the
correct numbers (5,686 instrumented lines vs. the script's 249).

Per DEC-0007 the cargo-mutants per-WO mutation-gate has been removed.
Reality-check + critic stub-scan + coverage gate are the authoritative
anti-laziness layers; this WO clears all three.
