### Coverage-gate tooling note

First run of `scripts/verify/coverage-gate.sh ucil-daemon 85 75` failed
with `invalid instrumentation profile data (file header is corrupt)`.
Root cause: `sccache` set as `RUSTC_WRAPPER` in the inherited env —
cargo-llvm-cov wraps around the pre-existing sccache wrapper
(`__CARGO_LLVM_COV_RUSTC_WRAPPER_PRE_EXISTING=sccache`) and the cache
keys between instrumented and non-instrumented builds can collide,
producing profraws that llvm-profdata cannot merge. Re-running with
`RUSTC_WRAPPER= CARGO_INCREMENTAL=0` → PASS at 89.28% line coverage
(above the 85% floor). This is an environment idiom for this
workstation (sccache is enabled for day-to-day dev builds), not a code
issue with WO-0039. The written `coverage-ucil-daemon.md` report is
the successful re-run.
