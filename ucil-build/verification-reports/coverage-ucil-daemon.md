## Why the auto-script fails

The auto-runner `scripts/verify/coverage-gate.sh` exits 1 unless the
inherited `RUSTC_WRAPPER=sccache` from the user's shell is unset
before `cargo llvm-cov`'s show-env wrapper runs (the wrapper
delegation chain breaks when sccache is in `RUSTC_WRAPPER`). The fix
is to invoke `cargo llvm-cov` directly with `RUSTC_WRAPPER` unset:

```
cd /home/rishidarkdevil/Desktop/ucil-wt/WO-0049
env -u RUSTC_WRAPPER cargo llvm-cov clean --workspace
```

## Coverage caveat: `g1_sources.rs` at 7.30%

`g1_sources.rs` is a NEW file landed by WO-0049 with the four
production-shape `G1Source` impls (`KgG1Source`, `SerenaG1Source`,
`AstGrepG1Source`, `DiagnosticsG1Source`) plus the `G1SourceFactory`.
Its coverage sits at 7.30% because the unit acceptance test
`server::test_find_references_tool` uses **local** `TestG1Source`
impls per DEC-0008 §4 (UCIL-internal trait-as-seam pattern), not the
production impls. This is by design — the production impls require:

- `KgG1Source`: a real `Arc<Mutex<KnowledgeGraph>>` fixture (out of
  scope for unit tests).
- `SerenaG1Source`: a real `Arc<dyn SerenaHoverClient>` (CLAUDE.md
  forbids mocking critical deps; integration testing through a docker
  fixture is the right harness layer).
- `AstGrepG1Source` and `DiagnosticsG1Source`: explicitly
  production-shape stubs per WO-0049 scope_in lines 14 (subitems c+d).
  Real MCP-channel + LSP-bridge wiring is deferred to follow-up WOs
  (WO-0049 scope_out lines 31-32).

The crate aggregate (88.57%) clears the 85% floor, so the gate passes.
