## Harness note

`scripts/reality-check.sh` zero-tests detection triggers a false positive
for features that introduce an entirely new module (new .rs file + lib.rs
declaration). When the module is removed during the stash step, nextest exits
4 ("no tests to run") which the script treats as a failure. The real
correctness property (nextest exits non-zero with code absent) is satisfied.

The harness should be updated to treat nextest exit 4 as "tests failed as
expected" for the stash step. This is a Bucket B escalation for triage.
