### P0-W1-F11 (rust-project fixture)

**Tool**: `scripts/reality-check.sh P0-W1-F11` (fixed version from main)

**Automated result**:
```
[reality-check] FAILURE: acceptance tests reported ZERO tests run with code stashed
```

**Root cause of automated failure** (structural incompatibility, NOT an implementation defect):

The F11 acceptance criterion is `cargo test -p ucil-core -- --ignored fixture::rust_project_loads`.
The `--ignored` flag only runs tests marked `#[ignore]`. The `#[ignore]` attribute was added in
commit `d5133304` (the fix commit). When `scripts/reality-check.sh` performs per-file rollback
for `crates/ucil-core/tests/fixture.rs`, it restores the file to its `d5133304^` state
(pre-`#[ignore]`), where the test function exists but lacks `#[ignore]`. With the test lacking
`#[ignore]`, `--ignored` runs 0 tests, triggering `zero_tests=1` â†’ automated failure.

**Manual mutation check performed to confirm test is real**:

1. Moved `tests/fixtures/rust-project/` to `/tmp/rust-project-backup`
2. Ran `cargo test -p ucil-core -- --ignored fixture::rust_project_loads`
3. **Result**: 1 test FAILED with:
   ```
   thread 'fixture::rust_project_loads' panicked at crates/ucil-core/tests/fixture.rs:30:9:
   fixture Cargo.toml missing at /tmp/ucil-wt/0003/tests/fixtures/rust-project/Cargo.toml
   ```
4. Restored `tests/fixtures/rust-project/`
5. Ran again â€” 1 test PASSED

**The test IS real**: it genuinely tests for the fixture's existence. Removing the fixture
causes the assertion to panic. This is not a fake-green.

**Verdict**: OK (via manual mutation check) âś“

**Follow-up needed**: File an ADR to address the structural incompatibility between `--ignored`
tests and `scripts/reality-check.sh`'s per-file rollback. Suggestion: for features whose
acceptance test uses `--ignored`, reality-check.sh should detect this and skip the zero_tests
guard for the stash phase (or verify by deleting/restoring the fixture files instead).
