`.rs|.ts|.tsx|.py` files outside `^tests/` and `^ucil-build/`; for both

("Running 0 tests" / "0 passed" → flagged as zero_tests=1).

For features that introduce an entirely **new** test (rather than
mutating production code reachable from existing tests), the reality
check's intuition does not apply — the test simply does not exist in the
parent state, which is the **correct** semantics for a new feature, not
a fake-green. Standing carry-forward per WO-0079 scope_out #18 →
WO-0080 verifier §Reality-check (informational) → WO-0084 verifier
section → WO-0085 verifier §Mutation checks (per-feature reality
