## Critic warnings — verifier confirmation

The critic report (`ucil-build/critic-reports/WO-0013.md`) flagged three
warnings for verifier confirmation. Dispositions:

1. **`.expect("len() == 1 above guarantees one element is available")` at
   `crates/ucil-cli/src/commands/plugin.rs:292`** — the expect is inside a
   `match matches.len() { … 1 => … }` arm where the matching arm statically
   guarantees `len() == 1`; clippy-pedantic+nursery is clean on the diff.
   `.claude/rules/rust-style.md` prefers `?` over `.expect()` in production
   paths, but the idiom here is invariant-guarded rather than error-masking.
   **Accepted as a non-blocking style nit**, not a rejection trigger.
   Recommend a future cleanup that destructures `matches.into_iter()` into
   a `(Some, None)` pair to satisfy the rule without the sentinel; not a
   gate.

2. **`aece7d8` edited the private `run_tools_list` helper despite the WO's
   scope_in marking `plugin_manager.rs` changes "additive only"** — the
   public API surface DID remain additive (`health_check_with_timeout` is
   the new `pub async fn`; the existing `health_check` is byte-rewired to
   delegate to it with the old default). The private helper fix was load-
   bearing: real Serena v1.0.0 rejects a bare `tools/list` with JSON-RPC
   error `-32602` because the MCP spec (2024-11-05) mandates the
   `initialize` → `notifications/initialized` round-trip first. Without
   this fix, the feature's real-server acceptance could not turn green.
   The mock-mcp-plugin was updated to the same protocol so
   `test_hot_cold_lifecycle` (regression guard) still passes. **Accepted
   as a required correctness fix scoped to a private helper**; the
   public-API invariant the planner gatekeeps is preserved.

3. **Commit `c3bfdc9` is 553 lines in one commit** — covered by DEC-0005
   (`module-coherence-commits`). Precedent matches WO-0006's `symbols.rs`
   (605 lines). No action.
