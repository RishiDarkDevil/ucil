## Disclosed deviations (acknowledged)

The RFR documents two pure-discovery deviations from WO-0076 scope_in,
neither expanding scope:

1. **Semgrep version pivot 0.9.0 → 0.8.1** — required because
   `semgrep-mcp@0.9.0` ships only `deprecation_notice` (live capture
   confirmed). Pinning would fail the F04 acceptance criterion (≥1
   security finding using the OWASP rule set). v0.8.1 is the last PyPI
   release with the canonical scan-tool surface. Manifest top-of-file
   rustdoc (lines 23-64) + RFR §"Live tool name capture" both document
   the pivot rationale.
2. **Semgrep CLI dependency surfaced as operator-state** — both
   v0.8.1 and v0.9.0 require the Semgrep CLI on PATH (or `SEMGREP_PATH`).
   The integration test + F04 verify script apply the WO-0069 §planner
   short-circuit pattern (originally an API-key gate, here adapted to
   a CLI-binary gate) — when no working Semgrep CLI is found, the
   smoke skips with `[SKIP]` and exit 0. The verifier's environment
   has semgrep CLI 1.159.0 on PATH so the load-bearing path was
   exercised end-to-end; no skip path was taken.
