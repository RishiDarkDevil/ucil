## Diff scope note (two-dot vs three-dot)

Run literally, this returns one extra file — `ucil-build/critic-reports/WO-0046.md` — because the critic landed its `1f0c089 critic(WO-0046): plugin-lifecycle integration suite — CLEAN` commit on `main` AFTER the feature branch was cut from `d5471b5` (the planner commit, which is the merge-base of `main` and HEAD). ```
$ git merge-base main HEAD

$ git log d5471b5..main --oneline
1f0c089 critic(WO-0046): plugin-lifecycle integration suite — CLEAN

The intended invariant — "the executor's branch only adds the WO-authorised paths" — is correctly captured by the three-dot diff `main...HEAD` (= "what HEAD added since divergence"), which returns exactly the 5 allow-listed paths. AC30 was re-run with the three-dot semantics and PASSES; the literal two-dot reading is a benign branch-graph artifact (executor never touched the critic report).
