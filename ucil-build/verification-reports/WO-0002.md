wiring commit) as the reference for ALL files. For types.rs, which was added
in an earlier commit (ea983dd), `git show e5ecae2^:types.rs` returns the full
types.rs content (since it already existed at that point). The rollback
effectively overwrites types.rs with itself -- no mutation occurs.

`pub mod types;`). With the types module undeclared, `cargo test -p ucil-core
types::` finds 0 matching tests and exits 0. The script interprets exit 0 as

`cargo test -p ucil-core types::` fails with exit 101 (E0583: file not found
for module `types`). Restored types.rs. Tests pass with 7 green. The
mutation property holds. The code is genuine.

**Root cause**: `scripts/reality-check.sh` needs per-file rollback references
instead of a single LAST_COMMIT. See escalation
`20260415-1630-reality-check-per-file-rollback.md`.
