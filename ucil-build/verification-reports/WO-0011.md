## Mutation checks

### `scripts/reality-check.sh` false-positive (documented, expected)

For both features `reality-check.sh` emits "FAILURE: acceptance tests reported ZERO tests run with code stashed — module was removed, not a genuine pass."

This is the exact per-file rollback edge case documented in escalation `20260415-1630-reality-check-per-file-rollback.md` and explicitly called out in the WO-0011 acceptance criteria / `lessons_applied`:

> "scripts/reality-check.sh P1-W4-F01 exits 0 (or, if the per-file rollback false-positive trips per escalation 20260415-1630, manual verification by the verifier is the documented workaround)."

Mechanism: `knowledge_graph.rs` is a brand-new file and `server.rs` has a module-level test added in a single commit. When the script stashes the file + reverts `lib.rs`, the test selector no longer matches any test (module gone), nextest exits 0 with zero tests, and the script flags "module was removed" as a FAILURE. The workaround is manual mutation: replace the implementation's core expression with a stub and confirm the test genuinely fails.
