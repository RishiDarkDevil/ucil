## Suspected cause

`notify` 6.x and `notify-debouncer-full` 0.3.x each spawn their own OS-level background threads (`notify`'s backend uses an inotify reader thread on Linux and `notify-debouncer-full` owns a separate dispatcher thread). Even when `watcher::*` tests are NOT being run, these library initializers are compiled into the `ucil-daemon` test binary and the threads may be created/torn down in ways that leave the LLVM profile runtime (which uses one profraw file per process) unable to flush a consistent header — producing the 0-byte and 472-byte truncated profraw files observed.

The fingerprint `557569589502809164` recurring across the 0-byte and corrupt files (and NOT across the valid-size files) strongly suggests **multiple processes with the same binary fingerprint** are competing to write the same profile file.

Verifier does not debug — flagging the most-likely cause is permitted, but the actual investigation belongs to the executor or a Bucket-D micro-WO.
