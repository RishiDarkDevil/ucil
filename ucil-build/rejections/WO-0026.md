## Scope-restriction confirms it's the dep-graph, not watcher tests

The llvm-cov failure persists even when **watcher tests are excluded** from
the run:

```
$ cargo llvm-cov nextest --package ucil-daemon -E 'not test(watcher::)'
...
     Summary [ 0.622s] 49 tests run: 49 passed, 6 skipped
warning: target/llvm-cov-target/WO-0026-777575-557569589502809164_7.profraw: invalid instrumentation profile data (file header is corrupt)
error: no profile can be merged
```

Same 0-byte / corrupt profraw files, same "no profile can be merged" error.
That rules out the watcher tests themselves as the cause and points at
something in the new dependency graph — most likely `notify` /
`notify-debouncer-full`'s OS-thread startup inside the ucil-daemon test
binary causing the LLVM profile runtime's per-process file to be written by
multiple threads sharing the same `%m` fingerprint and clobbering each other.

## Suspected cause (unchanged from retry 1)

`notify` 6.x and `notify-debouncer-full` 0.3.x each spawn their own OS-level
background threads (`notify`'s backend uses an inotify reader thread on Linux
and `notify-debouncer-full` owns a separate dispatcher thread). Even when
`watcher::*` tests are NOT being run, these library initializers are compiled
into the `ucil-daemon` test binary and the threads may be created/torn down
in ways that leave the LLVM profile runtime (which uses one profraw file per
process) unable to flush a consistent header — producing the 0-byte and
472-byte truncated profraw files observed.

The fingerprint `557569589502809164` recurring across the 0-byte and corrupt
files (and NOT across the valid-size files) strongly suggests **multiple
processes with the same binary fingerprint** are competing to write the same
profile file. This retry-2 run reproduces the exact same fingerprint, which
rules out any environmental randomness and reinforces that this is
deterministic behaviour of the new dep-graph.

Verifier does not debug — flagging the most-likely cause is permitted, but
the actual investigation belongs to the executor or a Bucket-D micro-WO.
