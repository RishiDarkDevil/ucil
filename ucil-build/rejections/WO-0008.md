## Suspected cause

`test_session_state_tracking` exercises each new method once with one
representative value. The assertions are "at least one entry present",
"count matches expected", "set contains X", "option is Some(X)" — none
of which exercise the arithmetic in `create_session::expires_at`, the
inequality in `purge_expired`, or the subtraction in the "how many
removed" return value. The test must drive a purge scenario where **some
sessions are past TTL and some are not**, then assert on the exact
remaining set, so that each operator in the code is independently
sensitive to a corresponding assertion.
