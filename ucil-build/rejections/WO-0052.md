## Suspected cause

`authority_rank` was added as a private `const fn` in
`67fd2eb feat(daemon): add fuse_g1 + authority_rank for G1 result fusion`
(WO-0048) but the rustdoc on `fuse_g1` / `G1Conflict` / `fields` (all
`pub`) references it via `[`authority_rank`]`. Under
`#![deny(warnings)]` (which implies
`#[deny(rustdoc::private_intra_doc_links)]`), this is a hard error.

Similarly `test_g1_parallel_execution` is referenced from a public
`///` comment at `executor.rs:952` but the symbol resolves to a
`#[cfg(test)]` item not present in the `--no-deps` rustdoc graph.

`run_g1_source` is referenced from `execute_g1`'s rustdoc but is a
private function.

WO-0048's verifier did not include `cargo doc -p ucil-daemon --no-deps`
as an AC. WO-0049 / WO-0050 / WO-0051 verifiers also did not. WO-0052's
planner is the first to add AC11 — exposing the latent bug.
